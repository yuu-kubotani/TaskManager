# 🚀 Task Manager

**「セキュリティが厳しい環境でも、最高の生産性を。」**

**Task Manager** は、外部のクラウドサービスに一切依存せず、完全オフライン（オンプレミス）で動作するWindows向けのオールインワン・タスク管理ツールです。

「便利なタスク管理ツールを使いたいけれど、会社の規定でクラウドが使えない」「ソフトの導入に管理者権限（インストール作業）が必要でハードルが高い」——そんな悩みを解決し、**日本のビジネスパーソンの生産性を底上げすること**を目的として開発されました。

**管理者権限不要**でPC上の好きなフォルダに配置するだけで、誰でもすぐに安全に使い始めることができます。また、ソースコードをGitHub上に公開することで、処理の裏側に不正がないという**高い透明性**を保ち、誰もが安心して利用・拡張できるオープンなソフトウェアを目指しています。

「タスク管理」「カレンダー」「タイムライン記録」「分析レポート」といった高度な機能を、あなたのローカルPC上でセキュアかつ軽快に実現します。

---

## ✨ 主な機能

- 📋 **リスト & カンバンボードによる直感的なタスク管理**
  - プロジェクトごとにタスクを整理。カンバンボードでのドラッグ＆ドロップによるステータス変更。
  - リスト表示では、優先度や期日による並び替え、期限超過タスクのハイライト警告機能を搭載。
- 📅 **カレンダー & タイムライン（タイムトラッキング）**
  - カレンダー上での予定（イベント）管理と、1日のスケジュールを俯瞰できるタイムライン表示。
  - タスクの「開始／停止」ボタンによるリアルタイムな時間記録や、タイムライン上でのドラッグ操作による後からの実績入力に対応。
- ⏱️ **高度なトラッキングサポート（ポモドーロ・非アクティブ検知）**
  - 長時間作業の警告や、ポモドーロ・テクニック（例: 25分作業＋5分休憩）に基づく通知機能を内蔵。
  - PCの放置（非アクティブ状態）やスリープを検知し、作業記録を自動的に一時停止・再開確認。
- 📊 **ネイティブチャートによる分析レポート**
  - 蓄積した作業実績から「プロジェクト・カテゴリ別の消費時間」や「タスクの平均完了日数（リードタイム）」を美しいグラフで可視化。
  - オフライン環境でも動作し、サマリーをCSVとしてエクスポート可能。
- 🔁 **複雑な定期タスク（ルーチン）とテンプレート生成**
  - 「毎週月曜と水曜」「毎月月末」などの高度な繰り返し条件を設定し、タスクを自動生成。
  - 定型的なプロジェクト構造や複数タスクのセットを「テンプレート」として保存し、ワンクリックで展開。
- 📎 **関連ファイル・URLのドラッグ＆ドロップ添付**
  - タスクやプロジェクトに関連するローカルファイル、フォルダ、WebサイトのURLを紐付け、アプリ内から直接プレビュー・起動。
- 🔔 **リマインダーとカレンダー連携 (ICS)**
  - 期日や予定時刻に合わせたポップアップ/バルーン通知機能。
  - iCalendar形式 (.ics) のインポート・エクスポートによる、外部カレンダーアプリとの連携。
- ⚙️ **徹底したローカル管理とカスタマイズ性**
  - すべてのデータをローカルに保存。自動バックアップや古いタスクの自動アーカイブ機能を完備。パスコードロック機能によるプライバシー保護。
  - 目に優しい「ダークモード」、色覚多様性に配慮した「色覚サポートモード」、ウィンドウ透明度の調整など、自分好みのUIにカスタマイズ可能。

## 📂 ファイル構成

保守性と拡張性を高めるため、モダンなディレクトリ構成を採用しています。

- `README.md` : アプリケーションの説明書（このファイル）
- `build.bat` : ワンクリックでコンパイルとアプリの起動を行うための起動スクリプト
- `organize_files.bat` : 古いファイルを削除し、名前を統一する整理用スクリプト
- `TaskManager.exe` : ビルド成功後に生成されるアプリ本体
- `Src/` : アプリケーションのコアとなる C# ソースコード群
  - `Program.cs` : アプリの起点（エントリーポイント）
  - `Forms/` : 画面（UI）を構成するコード（`FormMain.cs`, `FormReport.cs`, `FormSettings.cs` など全20画面）
  - `Models/` : データ構造を定義するクラス群
    - `AppSettings.cs` (設定データ)
    - `Models.cs` (タスク・プロジェクト・予定データ)
    - `UndoCommands.cs` (元に戻す機能のデータ)
  - `Services/` : データの保存・読み込み・バックアップ処理
    - `DataService.cs`
  - `Utils/` : 共通ユーティリティ
    - `DarkModeRenderer.cs` (ダークモード描画)
    - `NativeMethods.cs` (Win32 API連携)
    - `Prompt.cs` (文字入力ダイアログ)
    - `ThemeManager.cs` (配色・テーマ管理)
- `Tools/` : ビルド時に自動取得される C# コンパイラ (Roslyn) 等が格納される裏方フォルダ
- `Data/` : 実行時に自動生成・保存されるデータファイル群
  - `tasks.csv`, `projects.json`, `events.json`, `settings.json`, `time_logs.json` など

## 🛠️ 実行・ビルド方法

専用の開発環境（Visual Studioなど）をインストールしていなくても、Windows PC（10/11等）に標準搭載されている機能のみを使って、インターネットに一切接続せず**完全オフラインでワンクリックでビルドして起動**できます。

1. このプロジェクトのフォルダをPCの好きな場所に配置します。
2. **（初回のみ）** フォルダ内にある **`organize_files.bat`** をダブルクリックして実行します。
   - *※古いファイルの削除とソースコード名の統一が自動で行われます。*
3. フォルダ内にある **`build.bat`** をダブルクリックして実行します。
4. インターネット通信を行うことなく、Windows内蔵のC#コンパイラを利用してアプリが生成・起動します。

> **💡 2回目以降の起動について**
> 一度ビルドが成功した後は、生成された **`TaskManager.exe`** を直接ダブルクリックするだけでアプリを起動できます。
> （ソースコードを編集して最新版に更新したい場合のみ、再度 `build.bat` を実行してください）

> **⚠️ セキュリティ警告が出た場合**
> 初回実行時や、バッチファイルがコンパイラツールをダウンロードした際に、Windows (SmartScreen) 等から「Windows によって PC が保護されました」という青い警告画面が出ることがあります。
> これはご自身のPC内でソースコードから生成したばかりの新しいファイルであるためです。**「詳細情報」→「実行」** をクリックして起動してください。
